## Mission Statement

Build an **enterprise-grade software platform** for civil engineering firms — not a simple application — with the discipline, documentation, and architecture standards of a mature software company, so that it can serve as Andes Engineering's own operating system today and as a commercial SaaS product tomorrow.

## Project Goal

The objective is explicitly **not** to build a simple application. Every decision on this project is weighed against seven non-negotiable qualities:

1. **Scalability** — the platform must handle growth in tenants, projects, and data volume without architectural rewrites.
2. **Maintainability** — code and documentation must remain understandable and safely changeable years from now, by people who did not write the original version.
3. **Modularity** — products (Andes Core, Projects, CRM, Structures, Geo, BIM, AI, Analytics) must be able to evolve independently. See [VISION.md](VISION.md).
4. **Clean Architecture** — dependencies flow inward toward the domain; infrastructure and frameworks are replaceable details, not the foundation.
5. **Excellent User Experience** — UX is treated as an engineering concern, not a cosmetic layer applied at the end. See [engineering-philosophy.md](engineering-philosophy.md).
6. **Artificial Intelligence integration** — AI is a first-class capability across the platform, not a bolted-on feature. See [ai-principles.md](ai-principles.md).
7. **Engineering-first workflows** — the software mirrors how structural and geotechnical engineers actually work, rather than forcing generic project-management patterns onto engineering practice.

## Final Objective

The end goal is not simply to complete an application.

The end goal is to create a **professional engineering platform** with documentation, architecture, and development standards comparable to mature software companies — one where any AI coding assistant can understand the project from the handbook alone, without relying on chat history.

This handbook is intended to become the **single source of truth** for the entire platform.

## Mission vs. Vision

[VISION.md](VISION.md) is the destination: a 10-year platform that becomes the operating system for engineering consulting companies, and eventually a commercial product. This document, MISSION.md, is the standard we hold ourselves to on the way there — the qualities that must be true of the platform at every stage, not just at the end.
